Explication du jeu de données

Le jeu de données contient des transactions effectuées par des titulaires de cartes de crédit en septembre 2013, en Europe. Il comprend 492 fraudes sur un total de 284 807 transactions, ce qui le rend très déséquilibré.

Les caractéristiques V1, V2, ... V28 sont les composants principaux obtenus par Analyse en Composantes Principales (PCA), une technique de réduction de dimensions. Cela signifie que ces caractéristiques ont été extraites pour résumer l'information essentielle tout en réduisant la taille des données. Les seules caractéristiques non transformées sont le "Time" (temps) et le "Amount" (montant).

La caractéristique "Class" indique si la transaction est une fraude (1) ou non (0).

Explications des caractéristiques V1 et V20 :

V1 : Il s'agit de la première composante principale extraite à partir des caractéristiques d'origine. Comme les données sont transformées, la signification exacte de V1 n'est pas directement interprétable sans la transformation inverse, mais elle représente une combinaison linéaire des variables d'origine qui capture la variance maximale dans les données.

V20 : De même, V20 est une autre composante principale, représentant une autre combinaison linéaire de caractéristiques d'origine. Les valeurs de V20 peuvent être interprétées de manière similaire à celles de V1, mais elles dépendent des données spécifiques utilisées pour la transformation PCA. Cela peut indiquer une caractéristique comportementale ou temporelle, mais son interprétation exacte dépend du modèle de données.

Qu'est-ce que l'Analyse en Composantes Principales (PCA) ?

L'Analyse en Composantes Principales (PCA) est une méthode statistique utilisée pour réduire la dimensionnalité des données tout en préservant autant que possible la variance des données. Elle crée de nouvelles variables (appelées composantes principales) qui sont des combinaisons linéaires des variables originales. PCA est souvent utilisée pour simplifier les jeux de données complexes et aider à identifier les motifs sous-jacents tout en réduisant les redondances dans les données. Cette technique est particulièrement utile lorsque de nombreuses caractéristiques sont fortement corrélées, comme dans ce jeu de données de transactions financières.
